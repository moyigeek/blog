# 2025-07-15 Work Log

## Blog Refactoring (Hexo → Astro)

Completed full refactoring of moyigeek's personal blog from Hexo to Astro 5.

### What was done:
1. **Initialized Astro 5 project** with Tailwind CSS, TypeScript, and necessary integrations (sitemap, MDX, RSS)
2. **Migrated all 21 blog posts** from Hexo `source/_posts/` to Astro `src/content/posts/`
   - Converted Hexo `{% asset_img %}` tags to standard Markdown image syntax
   - Fixed broken `file:///` local image references
   - Normalized code block language names (C→c, RUST→rust, pyhton→python)
3. **Migrated all image assets** to `public/images/posts/` and `public/images/`
4. **Created clean, minimal UI** with:
   - Header with navigation + mobile menu + theme toggle
   - Post cards with date, tags, excerpt
   - Post layout with TOC sidebar, reading time, copyright notice, Giscus comments
   - Archives page (grouped by year)
   - Tags system (tag cloud + per-tag post list)
   - Gallery page (masonry layout + lightbox)
   - About page
   - 404 page
   - RSS feed
   - Sitemap
5. **Set up Cloudflare Pages CI** via GitHub Actions (`.github/workflows/deploy.yml`)
6. **Cleaned up old Hexo files** (config files, scaffolds, themes, migration scripts)
7. **Updated .gitignore, .nvmrc, README.md**

### Key technical decisions:
- Used two-file pagination (index.astro + [page].astro) due to Astro 5 paginate() behavior
- Created slugifyTag() utility to handle special chars in tag names
- Used CSS variables for theming (light/dark mode)
- Kept Giscus comments from original Hexo setup

### Math formula support added:
- Installed `remark-math` + `rehype-katex` + `katex` for LaTeX math rendering
- Configured `astro.config.mjs` with remark/rehype plugins, `strict: false` for Chinese chars in math
- Added KaTeX CSS CDN to BaseLayout head
- Added dark mode KaTeX color adaptation in global.css
- Supports both inline ($...$) and block ($$...$$) math formulas
- Existing articles with math: deepseek一面面经.md ($A_{3}^{3}=6$), 操作系统基础知识.md (multiple formulas)

### Background image + article blur:
- Added `back-light.jpg` / `back-dark.jpg` as fixed background in BaseLayout
- Three-layer structure: `.bg-layer` (image) + `.bg-overlay` (translucent) + `.content-layer` (z-10)
- Article pages (`data-article="true"`): background gets `blur(20px)` + stronger overlay for readability
- Non-article pages: background visible through lighter overlay (0.55 light / 0.7 dark)
- Fixed KaTeX CSS: changed from CDN (version mismatch) to local `import 'katex/dist/katex.min.css'`

### Code highlighting (Shiki dual-theme):
- Configured `shikiConfig.themes: { light: 'github-light', dark: 'github-dark' }` in astro.config.mjs
- Fixed `C++` → `cpp` in 2 markdown files (10 code blocks)
- CSS: `.dark .astro-code` uses `--shiki-dark` / `--shiki-dark-bg` variables, light mode uses default inline styles
